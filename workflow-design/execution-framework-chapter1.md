# 工作流执行框架设计文档

## 第一章：执行框架概述

### 1.1 设计目标

工作流执行框架是整个工作流系统的核心运行时组件，负责将配置层定义的工作流拓扑结构转化为实际的执行过程。本框架的设计目标如下：

#### 1.1.1 核心目标

| 目标 | 描述 |
|------|------|
| **可靠性** | 保证工作流执行的原子性和一致性，支持断点续执行和错误恢复 |
| **可扩展性** | 支持新节点类型的插件式扩展，无需修改核心引擎 |
| **高性能** | 支持大规模并发执行，充分利用多核资源 |
| **实时性** | 通过 WebSocket 实时推送执行状态，提供良好的用户体验 |
| **分布式** | 支持 Client 端执行机集群，实现跨网络分区的任务执行 |

#### 1.2.1 各层职责说明

| 层级 | 核心模块 | 职责描述 |
|------|----------|----------|
| **配置层** | WorkflowService | 工作流的创建、查询、更新、删除、发布 |
| **配置层** | NodeValidator | 验证节点配置的完整性和正确性 |
| **配置层** | CompatibilityChecker | Skill 变更时的兼容性检查 |
| **调度层** | WorkflowScheduler | 接收执行请求，分配执行资源，创建执行上下文 |
| **调度层** | ExecutionEngine | 流程编排，按拓扑顺序执行节点，管理并发 |
| **调度层** | ExecutionContext | 管理执行过程中的变量、节点输出、前置节点计算 |
| **调度层** | StateManager | 管理执行状态，持久化到数据库 |
| **调度层** | WebSocketManager | 管理 WebSocket 连接，推送执行状态 |
| **执行层** | NodeExecutor | 各类型节点的具体执行逻辑 |
| **执行层** | SkillExecutor | 调用 Skill 执行套件，处理分布式执行 |

#### 1.3.2 节点类型与执行器映射

| 节点类型 | 类型代码 | 分类 | 执行器类 | 说明 |
|----------|----------|------|----------|------|
| 开始节点 | start | BASIC | StartNodeExecutor | 工作流入口，初始化参数 |
| 结束节点 | end | BASIC | EndNodeExecutor | 工作流出口，输出结果 |
| 简单条件 | condition_simple | LOGIC | SimpleConditionExecutor | 二分支条件判断 |
| 多路条件 | condition_multi | LOGIC | MultiConditionExecutor | 多分支条件判断 |
| 循环节点 | loop | LOGIC | LoopNodeExecutor | 顺序遍历执行 |
| 批处理节点 | batch | LOGIC | BatchNodeExecutor | 并发批量执行 |
| 异步处理节点 | async | LOGIC | AsyncNodeExecutor | 异步执行子工作流 |
| 结果收集节点 | collect | LOGIC | CollectNodeExecutor | 收集异步任务输出 |
| 技能节点 | skill | EXECUTION | SkillNodeExecutor | 执行 Skill 库中的技能 |

### 1.5 设计原则

#### 1.5.1 SOLID 原则应用

| 原则 | 应用场景 |
|------|----------|
| **单一职责 (SRP)** | 每个节点执行器只负责一种节点类型的执行 |
| **开闭原则 (OCP)** | 新增节点类型只需实现 NodeExecutor 接口，无需修改引擎 |
| **里氏替换 (LSP)** | 所有执行器可替换使用，遵循统一接口 |
| **接口隔离 (ISP)** | NodeExecutor 接口精简，只包含必要方法 |
| **依赖倒置 (DIP)** | 引擎依赖抽象接口，不依赖具体实现 |

#### 1.5.2 执行保证

```
┌─────────────────────────────────────────────────────────────────┐
│                        执行保证机制                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  原子性 (Atomicity)                                      │   │
│  │  - 节点执行状态变更与数据库持久化同步                     │   │
│  │  - 使用事务保证状态一致性                                 │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  可恢复性 (Recoverability)                               │   │
│  │  - 执行状态持久化，支持中断后恢复                         │   │
│  │  - 节点输出缓存，支持重新执行时跳过已完成节点             │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  幂等性 (Idempotency)                                    │   │
│  │  - 相同输入的节点执行产生相同结果                         │   │
│  │  - 重复执行不会产生副作用                                 │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  隔离性 (Isolation)                                      │   │
│  │  - 不同工作流执行相互隔离                                 │   │
│  │  - 执行上下文独立，变量不共享                             │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```
