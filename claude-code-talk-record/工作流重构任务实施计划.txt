● 设计第三部分：验证逻辑设计

  验证器架构

  validation/
  ├── WorkflowValidator.java          # 验证入口，协调各验证器
  ├── StructureValidator.java         # 结构验证
  ├── ParameterReferenceValidator.java # 参数引用验证
  ├── CyclicDependencyValidator.java   # 循环依赖检测
  └── SkillCompatibilityChecker.java   # Skill 兼容性检查

  各验证器职责

  StructureValidator（结构验证）：
  - 检查是否存在且仅存在一个开始节点
  - 检查是否存在且仅存在一个结束节点
  - 检查是否存在孤立节点（未连接任何其他节点）
  - 检查所有必填参数是否已配置

  ParameterReferenceValidator（参数引用验证）：
  - 解析 ${节点名.参数名} 格式的引用
  - 验证被引用的节点是否存在
  - 验证被引用的参数是否存在于目标节点的输出参数中
  - 验证被引用的节点是否为当前节点的前置节点

  CyclicDependencyValidator（循环依赖检测）：
  - 构建工作流的 DAG 图
  - 使用 Kahn 算法或 DFS 检测环
  - 返回检测结果和涉及的节点

  SkillCompatibilityChecker（Skill 兼容性检查）：
  - 比较 Skill 当前版本与节点保存的快照
  - 检测新增必填参数 → NEEDS_UPDATE
  - 检测参数类型变更 → INCOMPATIBLE
  - 检测参数删除 → INCOMPATIBLE
  - Skill 被删除 → INVALID
  - 更新节点的 compatibility_status 字段

  Skill 变更监听

  @Component
  public class SkillChangeListener {

      @EventListener
      public void onSkillUpdated(SkillUpdatedEvent event) {
          // 查找所有使用该 Skill 的节点
          // 执行兼容性检查
          // 更新节点的 compatibility_status
      }
  }
