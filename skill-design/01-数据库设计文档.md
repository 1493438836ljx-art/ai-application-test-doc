# Skill库数据库设计文档

## 1. 数据库概述

### 1.1 数据库信息
- **数据库类型**: MySQL 8.0+
- **字符集**: utf8mb4
- **排序规则**: utf8mb4_unicode_ci
- **存储引擎**: InnoDB

### 1.2 表清单

| 表名 | 说明 | 关联 |
|------|------|------|
| skill | Skill主表 | - |
| skill_parameter | 参数表 | → skill.id |
| skill_access_control | 访问控制表 | → skill.id |

## 2. 表结构设计

### 2.1 skill (Skill主表)

```sql
CREATE TABLE IF NOT EXISTS skill (
    id VARCHAR(36) NOT NULL COMMENT 'Primary key, UUID',
    name VARCHAR(100) NOT NULL COMMENT 'Skill name, globally unique',
    description VARCHAR(2000) DEFAULT NULL COMMENT 'Skill description',
    suite_path VARCHAR(500) DEFAULT NULL COMMENT 'Executable suite file path',
    suite_filename VARCHAR(500) DEFAULT NULL COMMENT 'Executable suite file name (with extension)',
    execution_type VARCHAR(20) NOT NULL COMMENT 'Execution type: AUTOMATED/AI',
    category VARCHAR(20) NOT NULL COMMENT 'Category: SYSTEM/USER',
    access_type VARCHAR(20) NOT NULL COMMENT 'Access control: PUBLIC/PRIVATE/WHITELIST/PROJECT',
    is_container TINYINT(1) NOT NULL DEFAULT 0 COMMENT 'Is container: 0=false, 1=true',
    allow_add_input_params TINYINT(1) DEFAULT 0 COMMENT 'Allow add input parameters dynamically',
    allow_add_output_params TINYINT(1) DEFAULT 0 COMMENT 'Allow add output parameters dynamically',
    status VARCHAR(20) NOT NULL COMMENT 'Status: PUBLISHED/DRAFT',
    created_by VARCHAR(100) NOT NULL COMMENT 'Creator',
    updated_by VARCHAR(100) NOT NULL COMMENT 'Updater',
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT 'Created at',
    updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT 'Updated at',
    deleted TINYINT(1) NOT NULL DEFAULT 0 COMMENT 'Logical deletion flag: 0=false, 1=true',
    deleted_at DATETIME DEFAULT NULL COMMENT 'Deletion time (for aging cleanup)',
    PRIMARY KEY (id),
    UNIQUE INDEX idx_name (name),
    INDEX idx_category (category),
    INDEX idx_status (status),
    INDEX idx_created_by (created_by),
    INDEX idx_deleted_at (deleted_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='Skill main table';
```

#### 字段详细说明

| 字段名 | 数据类型 | 可空 | 默认值 | 说明 |
|--------|----------|------|--------|------|
| id | VARCHAR(36) | NO | - | UUID主键，由应用层生成 |
| name | VARCHAR(100) | NO | - | Skill名称，全局唯一，用于标识和搜索 |
| description | VARCHAR(2000) | YES | NULL | Skill详细描述，支持多行文本 |
| suite_path | VARCHAR(500) | YES | NULL | 执行套件文件在服务器上的存储路径 |
| suite_filename | VARCHAR(500) | YES | NULL | 执行套件原始文件名(用于下载时显示) |
| execution_type | VARCHAR(20) | NO | - | 执行类型枚举: AUTOMATED(自动化), AI(AI驱动) |
| category | VARCHAR(20) | NO | - | 分类枚举: SYSTEM(系统), USER(用户自定义) |
| access_type | VARCHAR(20) | NO | - | 访问控制类型: PUBLIC, PRIVATE, WHITELIST, PROJECT |
| is_container | TINYINT(1) | NO | 0 | 是否为容器类型Skill |
| allow_add_input_params | TINYINT(1) | YES | 0 | 是否允许动态添加入参 |
| allow_add_output_params | TINYINT(1) | YES | 0 | 是否允许动态添加出参 |
| status | VARCHAR(20) | NO | - | 状态枚举: PUBLISHED(已发布), DRAFT(草稿) |
| created_by | VARCHAR(100) | NO | - | 创建人标识(用户ID或用户名) |
| updated_by | VARCHAR(100) | NO | - | 最后更新人标识 |
| created_at | DATETIME | NO | CURRENT_TIMESTAMP | 创建时间，自动填充 |
| updated_at | DATETIME | NO | CURRENT_TIMESTAMP | 更新时间，自动更新 |
| deleted | TINYINT(1) | NO | 0 | 逻辑删除标记: 0=未删除, 1=已删除 |
| deleted_at | DATETIME | YES | NULL | 删除时间戳，用于老化清理计算 |

#### 索引设计

| 索引名 | 索引类型 | 字段 | 说明 |
|--------|----------|------|------|
| PRIMARY | 主键索引 | id | 聚簇索引 |
| idx_name | 唯一索引 | name | 保证名称唯一性，加速名称搜索 |
| idx_category | 普通索引 | category | 加速分类筛选 |
| idx_status | 普通索引 | status | 加速状态筛选 |
| idx_created_by | 普通索引 | created_by | 加速创建人筛选 |
| idx_deleted_at | 普通索引 | deleted_at | 加速老化清理查询 |

### 2.2 skill_parameter (参数表)

```sql
CREATE TABLE IF NOT EXISTS skill_parameter (
    id VARCHAR(36) NOT NULL COMMENT 'Primary key, UUID',
    skill_id VARCHAR(36) NOT NULL COMMENT 'Associated Skill ID',
    param_direction VARCHAR(10) NOT NULL COMMENT 'Parameter direction: INPUT/OUTPUT',
    param_order INT NOT NULL COMMENT 'Parameter order (starting from 1)',
    param_type VARCHAR(50) DEFAULT NULL COMMENT 'Parameter type',
    param_name VARCHAR(100) DEFAULT NULL COMMENT 'Parameter name',
    default_value VARCHAR(1000) DEFAULT NULL COMMENT 'Default value (only for input params)',
    description VARCHAR(500) DEFAULT NULL COMMENT 'Parameter description',
    required TINYINT(1) DEFAULT 0 COMMENT 'Is required: 0=false, 1=true',
    PRIMARY KEY (id),
    INDEX idx_skill_id (skill_id),
    INDEX idx_skill_direction (skill_id, param_direction),
    CONSTRAINT fk_param_skill FOREIGN KEY (skill_id) REFERENCES skill(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='Skill parameter table';
```

#### 字段详细说明

| 字段名 | 数据类型 | 可空 | 默认值 | 说明 |
|--------|----------|------|--------|------|
| id | VARCHAR(36) | NO | - | UUID主键 |
| skill_id | VARCHAR(36) | NO | - | 关联的Skill ID，级联删除 |
| param_direction | VARCHAR(10) | NO | - | 参数方向: INPUT(入参), OUTPUT(出参) |
| param_order | INT | NO | - | 参数顺序，从1开始，用于保持参数顺序 |
| param_type | VARCHAR(50) | YES | NULL | 参数类型，支持复合类型如 Array<String> |
| param_name | VARCHAR(100) | YES | NULL | 参数名称 |
| default_value | VARCHAR(1000) | YES | NULL | 默认值(仅入参使用) |
| description | VARCHAR(500) | YES | NULL | 参数描述说明 |
| required | TINYINT(1) | YES | 0 | 是否必填: 0=可选, 1=必填 |

#### 参数类型系统

**基本类型**:
| 类型 | 说明 | 示例值 |
|------|------|--------|
| String | 字符串 | "hello world" |
| Boolean | 布尔值 | true, false |
| Integer | 整数 | 123 |
| Object | JSON对象 | {"key": "value"} |
| Times | 时间类型 | "2024-01-01 00:00:00" |

**复合类型**:
| 类型格式 | 说明 | 示例 |
|----------|------|------|
| Array<ElementType> | 数组类型 | Array<String> |
| File<FileType> | 文件类型 | File<Zip> |
| Array<File<FileType>> | 文件数组 | Array<File<Excel>> |

**文件子类型**:
- Zip: ZIP压缩文件
- Doc: Word文档(.doc)
- Docx: Word文档(.docx)
- Excel: Excel文件(.xlsx, .xls)
- Pdf: PDF文件
- Txt: 文本文件

#### 索引设计

| 索引名 | 索引类型 | 字段 | 说明 |
|--------|----------|------|------|
| PRIMARY | 主键索引 | id | 聚簇索引 |
| idx_skill_id | 普通索引 | skill_id | 加速按Skill查询 |
| idx_skill_direction | 复合索引 | skill_id, param_direction | 加速按Skill和方向查询 |

### 2.3 skill_access_control (访问控制表)

```sql
CREATE TABLE IF NOT EXISTS skill_access_control (
    id VARCHAR(36) NOT NULL COMMENT 'Primary key, UUID',
    skill_id VARCHAR(36) NOT NULL COMMENT 'Associated Skill ID',
    target_type VARCHAR(20) NOT NULL COMMENT 'Target type: USER/PROJECT',
    target_id VARCHAR(36) NOT NULL COMMENT 'Target ID',
    PRIMARY KEY (id),
    INDEX idx_skill_id (skill_id),
    INDEX idx_target (target_type, target_id),
    UNIQUE INDEX idx_skill_target (skill_id, target_type, target_id),
    CONSTRAINT fk_access_control_skill FOREIGN KEY (skill_id) REFERENCES skill(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='Skill access control table';
```

#### 字段详细说明

| 字段名 | 数据类型 | 可空 | 默认值 | 说明 |
|--------|----------|------|--------|------|
| id | VARCHAR(36) | NO | - | UUID主键 |
| skill_id | VARCHAR(36) | NO | - | 关联的Skill ID，级联删除 |
| target_type | VARCHAR(20) | NO | - | 目标类型: USER(用户), PROJECT(项目) |
| target_id | VARCHAR(36) | NO | - | 目标ID(用户ID或项目ID) |

#### 访问控制类型说明

| access_type | 说明 | access_control表使用 |
|-------------|------|---------------------|
| PUBLIC | 公开，所有用户可访问 | 不需要记录 |
| PRIVATE | 私有，仅创建者可访问 | 不需要记录 |
| WHITELIST | 白名单，指定用户可访问 | target_type=USER |
| PROJECT | 项目级，项目成员可访问 | target_type=PROJECT |

#### 索引设计

| 索引名 | 索引类型 | 字段 | 说明 |
|--------|----------|------|------|
| PRIMARY | 主键索引 | id | 聚簇索引 |
| idx_skill_id | 普通索引 | skill_id | 加速按Skill查询 |
| idx_target | 复合索引 | target_type, target_id | 加速按目标查询 |
| idx_skill_target | 唯一索引 | skill_id, target_type, target_id | 防止重复授权 |

## 3. 数据库迁移历史

### 3.1 V1.3__add_skill_library.sql

**版本**: 1.3
**说明**: 创建Skill库初始表结构

**变更内容**:
- 创建skill主表
- 创建skill_input_parameter入参表
- 创建skill_output_parameter出参表
- 创建skill_access_control访问控制表

**原始设计**: 入参和出参使用独立的表

### 3.2 V1.4__update_skill_library_schema.sql

**版本**: 1.4
**说明**: 更新Skill库Schema

**变更内容**:
- skill表添加allow_add_input_params字段
- skill表添加allow_add_output_params字段
- skill_input_parameter表删除variadic字段
- skill_output_parameter表删除variadic字段

### 3.3 V2024.01.01__merge_skill_parameter_tables.sql

**版本**: 2024.01.01
**说明**: 合并入参和出参表

**变更内容**:
- 创建新的统一参数表skill_parameter
- 添加param_direction字段区分入参/出参
- 迁移skill_input_parameter数据到skill_parameter
- 迁移skill_output_parameter数据到skill_parameter
- 删除旧表skill_input_parameter和skill_output_parameter

**迁移SQL**:
```sql
-- 迁移入参数据
INSERT INTO skill_parameter (id, skill_id, param_direction, param_order, param_type, param_name, default_value, description, required)
SELECT id, skill_id, 'INPUT', param_order, param_type, param_name, default_value, description, IFNULL(required, 0)
FROM skill_input_parameter;

-- 迁移出参数据
INSERT INTO skill_parameter (id, skill_id, param_direction, param_order, param_type, param_name, default_value, description, required)
SELECT id, skill_id, 'OUTPUT', param_order, param_type, param_name, NULL, description, IFNULL(required, 0)
FROM skill_output_parameter;
```

## 4. 查询设计

### 4.1 常用查询

#### 4.1.1 分页搜索Skill

```sql
SELECT * FROM skill
WHERE deleted = 0
  AND (:name IS NULL OR name LIKE CONCAT('%', :name, '%'))
  AND (:category IS NULL OR category = :category)
  AND (:status IS NULL OR status = :status)
  AND (:createdBy IS NULL OR created_by = :createdBy)
ORDER BY created_at DESC
LIMIT :size OFFSET :offset;
```

#### 4.1.2 获取Skill详情(含参数)

```sql
-- 主信息
SELECT * FROM skill WHERE id = :id AND deleted = 0;

-- 入参
SELECT * FROM skill_parameter
WHERE skill_id = :id AND param_direction = 'INPUT'
ORDER BY param_order ASC;

-- 出参
SELECT * FROM skill_parameter
WHERE skill_id = :id AND param_direction = 'OUTPUT'
ORDER BY param_order ASC;

-- 访问控制
SELECT * FROM skill_access_control WHERE skill_id = :id;
```

#### 4.1.3 统计参数数量

```sql
SELECT
  COUNT(CASE WHEN param_direction = 'INPUT' THEN 1 END) as input_count,
  COUNT(CASE WHEN param_direction = 'OUTPUT' THEN 1 END) as output_count
FROM skill_parameter
WHERE skill_id = :id;
```

#### 4.1.4 老化清理查询

```sql
-- 查询已删除超过365天的Skill
SELECT * FROM skill
WHERE deleted = 1
  AND deleted_at IS NOT NULL
  AND deleted_at < DATE_SUB(NOW(), INTERVAL 365 DAY);
```

## 5. 数据完整性

### 5.1 外键约束

| 子表 | 外键字段 | 父表 | 级联删除 |
|------|----------|------|----------|
| skill_parameter | skill_id | skill | CASCADE |
| skill_access_control | skill_id | skill | CASCADE |

### 5.2 唯一约束

| 表名 | 约束字段 | 说明 |
|------|----------|------|
| skill | name | Skill名称全局唯一 |
| skill_access_control | skill_id + target_type + target_id | 同一Skill不能重复授权 |

### 5.3 非空约束

**skill表必填字段**:
- id, name, execution_type, category, access_type, is_container, status
- created_by, updated_by, created_at, updated_at, deleted

**skill_parameter表必填字段**:
- id, skill_id, param_direction, param_order

**skill_access_control表必填字段**:
- id, skill_id, target_type, target_id

## 6. 数据生命周期

### 6.1 创建流程

```
1. 插入skill记录 (status=DRAFT, deleted=0)
2. 批量插入skill_parameter记录
3. 批量插入skill_access_control记录(如有)
```

### 6.2 更新流程

```
1. 更新skill主表字段
2. 删除旧的入参 → 插入新的入参
3. 删除旧的出参 → 插入新的出参
4. 访问控制(暂不更新)
```

### 6.3 删除流程(逻辑删除)

```
1. 删除skill_parameter关联记录
2. 删除skill_access_control关联记录
3. 删除执行套件文件(物理删除)
4. 设置skill.deleted=1, skill.deleted_at=NOW()
```

### 6.4 物理删除(老化清理)

```
1. 定时任务每天凌晨0:05执行
2. 查询deleted=1且deleted_at超过365天的记录
3. 物理删除skill_parameter记录
4. 物理删除skill_access_control记录
5. 物理删除skill记录
```

## 7. 性能优化建议

### 7.1 索引优化

- 已建立的索引覆盖了常用查询场景
- 如有大量按名称搜索需求，可考虑添加全文索引

### 7.2 分页优化

- 使用覆盖索引减少回表
- 对于深分页，考虑使用游标分页

### 7.3 查询优化

- 列表查询不返回参数详情，仅返回计数
- 详情查询才关联查询参数表
